| 对比项                         | `interface`                  | `type`（类型别名）                 |
| ------------------------------ | ---------------------------- | ---------------------------------- |
| **能否扩展（Extend）**         | ✅ 可以 `extends`            | ✅ 可以 `extends`                  |
| **能否被扩展**                 | ✔ 可以被再次声明（接口合并） | ❌ 不能重复声明同名 type           |
| **能否映射（Mapped Types）**   | ❌                           | ✅ 支持：`{ [K in ...]: ... }`     |
| **能否表达联合/交叉/原始类型** | ❌                           | ✅ 可以（`string                   |
| **语义定位**                   | 更偏向“对象结构规范”         | 更偏向“类型表达式”                 |
| **是否推荐用于库、公共 API**   | ✅ 更推荐（可扩展）          | 仅结构定死时推荐                   |
| **是否能用于高阶类型套路**     | 一般                         | ✅ 更常用（infer、联合、工具类型） |

> `interface` 支持 自动合并，非常适合：
>
> - 第三方库类型扩展
> - 大型项目想让模块补充类型时

## 用 interface:

- 设计对外暴露的 API、SDK、库
- 需要可扩展
- 需要让多人模块补充同一个结构
- 表示对象结构、类结构

## 用 type：

- 需要构造复杂类型
- 联合、交叉、条件、元组
- 实现工具类型
- 联合字符串、模板字符串
- 类型操作逻辑（注重类型推导）
